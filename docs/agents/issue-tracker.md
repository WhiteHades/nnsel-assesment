# Issue tracker

This repository uses GitHub Issues.

Use the `gh` CLI for issue creation and lookup when local work needs issue context.

## Repository

- Owner: `WhiteHades`
- Repository: `nnsel-assesment`
- Remote: `origin`

## Expected use

- Planning issues can be created after the GitHub repository exists.
- Implementation issues should map to one feature slice each.
- Each issue should include scope, acceptance criteria, and test notes.

## Suggested issue slices

1. Docker and Odoo module scaffold
2. Fund accounts and incoming funds
3. Ledger-backed balance movements
4. Allocation workflow and approvals
5. Requisitions and bill control
6. Transfers between projects and expense heads
7. Security rules and multi-company access
8. Dashboard, notifications, and demo data
9. README, architecture notes, and recording script
